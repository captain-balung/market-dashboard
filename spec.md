## A. 總則區

### 專案精神

這個看板是為**單一使用者（作者本人）**打造的個人決策輔助工具，不是產品。它追求的是**「打開就懂、每天看一眼就掌握市場」**，不是功能大而全。

核心信念有三，每一條都可被引用來拒絕方案：

1. **服務一個人的習慣優於通用最佳實踐**——任何「為了更多使用者而增加的複雜度」都違反精神。沒有第二個使用者要討好。
2. **解讀優於轉貼**——新聞區存在的理由是「知道為何漲跌」，不是聚合連結牆。任何「只貼標題不解讀」的方案違反精神。
3. **兩種節奏分離**——「每日慢看」與「盯盤快看」是兩種心智模式，混在一頁會兩邊都做不好。任何「把儀表元件塞進新聞首頁」或反之的方案違反精神。

> 反面引用範例：「在首頁加即時跳動的 K 線」違反第 3 條——盯盤屬於儀表頁，首頁是定格慢看。

### 總體規範

以下 5 條工作原則，每條都可被 AI 用於拒絕方案：

1. **解讀先於聚合**：新聞功能的產出必須是 AI 重寫的繁中摘要 + 解讀，不接受原文整段翻譯貼上或純連結列表。
2. **定格優於即時**：首頁價格漲跌以「昨日 9:00 → 今日 9:00」定格為準；不為了「更即時」而引入 websocket 或秒級輪詢。違反者以「過度工程、違反精神第 1 條」拒絕。
3. **容錯不可拖垮全頁**：任一外部資料源（尤其 CNN 非官方端點）失效時，該區塊優雅降級顯示「暫無資料」，不得使整頁崩潰或卡載入。
4. **機器優先**：可自動化且可逆的操作（git、build、起 dev server、查 log、Supabase migration、Vercel 部署查詢、視覺截圖比對），AI 自行用 CLI/MCP 完成並驗證，不丟回人類手工。連續 2 次自動化失敗才升級給人類。
5. **唯讀公開、單點可編輯**：全站對訪客唯讀；僅 `/admin` 登入後可改自選標的。任何「讓訪客能改資料」的方案違反此條。

### 開發順序（管線優先）

順序**不可顛倒**：

① **部署優先**——專案初始化後第一時間就把空殼部署到 Vercel，綁定 Supabase，確認線上可訪問。不要等功能做完才接部署。

② **完整測通**——打通 git → Supabase → Next.js 後端 API route → 前端頁面 整條鏈路：能從資料庫讀一筆假資料顯示在線上頁面。

③ **最小可行產品**——先做出「首頁能顯示一個寫死標的的現價」這個最小端到端可見頁面，確認整條會動，再開始長五類新聞、儀表頁等功能。

**精神**：先讓水管通水，再裝水龍頭。

**配套人機節奏**：前期人類密集投入（定方向、確認管線與地基、給各 API key），後期偶爾參與（抽查、驗收、品味回饋）。AI 全程執行。分工按**時間軸階段**劃分，非按前端／後端。

## B. 產品定義區

### 核心命題

一個個人專用的市場看板：**首頁**每天早上自動呈現三大市場（加密／貴金屬／美股）的 AI 繁中新聞摘要與漲跌定格，**儀表頁**提供 TradingView K 線與情緒／總經儀表供盯盤細看。

### 目標使用者

**唯一使用者：作者本人。**

- 技術背景：能讀懂價格、技術指標、財經新聞；能進 Supabase 後台、Vercel 後台。
- 使用情境：(a) 每天早上打開首頁掃一遍當日市場；(b) 想細看某標的時進儀表頁看 K 線與情緒。
- 訪客：可能把網址分享給少數朋友，他們是**唯讀**訪客，不參與編輯。

### 範圍邊界

**做什麼：**

- 追蹤固定一組自選標的（美股七雄、6 個幣、PAXG 金、KAG 銀），可由 admin 增減
- 每日 09:00 生成五類新聞繁中摘要、每週一生成週報
- 價格牆（現價 + 9:00 定格漲跌）
- 儀表頁：TradingView K 線、3 個恐懼貪婪指數、5 項總經數據
- admin 登入後增減標的

**明確不做什麼（擋 scope creep）：**

- ❌ **不做下單／交易**——純看板，永不接券商或交易所下單 API
- ❌ **不做投資建議**——AI 只解讀「為何漲跌」，不產生買賣建議
- ❌ **不做即時報價／秒級推播**——首頁定格、現價 15 分輪詢已是上限
- ❌ **不做警報通知**——不做價格到價提醒、不發 email/推播
- ❌ **不做多使用者系統**——無註冊、無個人化清單，全站單一全域清單
- ❌ **不做自選標的的歷史走勢圖（sparkline）**——卡片只有現價＋漲跌%
- ❌ **不做行動 App**——響應式網頁即可，不出原生 App
- ❌ **不做新聞全文轉載**——僅 AI 重寫摘要 + 原文連結

### 成功標準（可驗收）

1. 早上 09:05 打開首頁，能看到當日五類新聞各約 3 則的繁中摘要，每則可點到原文；價格牆顯示 16 個標的的現價與「昨日→今日」漲跌%（美股休市標「休市」）。
2. 進入儀表頁點任一支援標的，3 秒內載入該標的 TradingView 圖（日線主圖掛滿指標 + 4h/1h/15m 小圖），並看到 3 個情緒儀表與 5 項總經數據（失效項顯示「暫無資料」而非整頁壞）。
3. 以 admin 帳密登入 `/admin`，新增一個標的後，首頁價格牆於下次載入即出現該標的；登出狀態下訪問 `/admin` 被導向登入頁。

## C. 功能與資安區

### 工程紅線（絕對禁止的工程行為）

1. **不准把任何 API key、admin 密碼、CRON_SECRET 寫死在程式碼或前端**——一律走環境變數，且機密 key 只在 server 端使用。
2. **不准 catch 後吞例外**——外部 API 失敗要記錄並回傳明確降級狀態（給前端顯示「暫無資料」），不得靜默吞掉。
3. **不准在前端暴露 server-only key**——CoinGecko/Finnhub/Marketaux/FRED/CryptoPanic/Anthropic 等呼叫一律在 Next.js server 端（API route / server component），前端只拿結果。
4. **不准讓單一資料源失效拖垮整頁**——每個外部呼叫要有 try/catch + 降級值，頁面其餘部分照常渲染。
5. **不准繞過 admin 認證直接寫入標的清單**——所有寫入 watchlist 的 API route 必須先驗證 admin session。

### 功能清單

| ID | 功能 | 驗收條件（可機器驗證） |
|---|---|---|
| **F-01** | 自選標的資料模型 | Supabase `watchlist` 表存在，含 16 筆種子資料（含 symbol、類別、TradingView symbol、是否有圖）；API route `GET /api/watchlist` 回傳 JSON 陣列且筆數=啟用中標的數 |
| **F-02** | 加密＋貴金屬價格抓取 | server 端呼叫 CoinGecko，輸入 6 幣+PAXG+KAG，回傳每個標的的現價（USD）；單元測試 mock 回應解析正確 |
| **F-03** | 美股價格抓取 | server 端呼叫 Finnhub，輸入 7 檔，回傳現價；休市時 `marketStatus=closed` 旗標為 true |
| **F-04** | 9:00 價格快照 | 每日 09:00（台北）寫入 `price_snapshot` 表一筆/標的；表中存在當日 09:00 時間戳的快照且筆數=啟用標的數 |
| **F-05** | 漲跌% 定格計算 | 給定昨日與今日 09:00 快照，API 回傳 `changePct = (今-昨)/昨`；美股無昨日交易快照時回傳 `closed` 而非數字 |
| **F-06** | 價格牆現價輪詢 | 首頁前端每 15 分鐘重新抓現價；現價與 9:00 定格漲跌% 同時顯示於卡片 |
| **F-07** | 加密新聞抓取 | server 呼叫 CryptoPanic 取前一日加密/貴金屬相關新聞，回傳標準化文章陣列（標題、連結、時間、來源） |
| **F-08** | 美股新聞抓取 | server 呼叫 Finnhub company-news + general news 取前一日，回傳標準化文章陣列 |
| **F-09** | 宏觀新聞抓取 | server 呼叫 Marketaux 以關鍵字（Fed/inflation/geopolitics 等）取前一日國際/總經新聞，回傳標準化文章陣列 |
| **F-10** | AI 每日摘要生成 | 每日 09:00 將五類前一日新聞分別送 Claude Sonnet，產出每類約 3 則繁中摘要（重寫非翻譯）+ 各附原文連結，寫入 `daily_digest` 表；單筆 digest 含 5 類、每類 ≤ 約 3 則、每則有 source_url |
| **F-11** | AI 週報生成 | 每週一 09:00 讀當周 7 份 `daily_digest`，送 Claude Sonnet 濃縮成週報，寫入 `weekly_digest` 表 |
| **F-12** | 首頁渲染 | `/` 顯示五類摘要區 + 價格牆 + 週報入口；緊湊密度；深淺主題可切換（預設深色），切換狀態存於 cookie/localStorage |
| **F-13** | 儀表頁 K 線區 | `/dashboard` 點側欄標的→主圖載入該標的日線 TradingView widget（掛 MA20/60/120、MACD、RSI、布林、量）+ 4h/1h/15m 三小圖（僅 MA）；KAG 顯示「無圖表」 |
| **F-14** | 商品列表側欄 | 儀表頁側欄列出所有啟用標的，點擊切換主圖標的；當前選中項有視覺標示 |
| **F-15** | 加密恐懼貪婪儀表 | 呼叫 Alternative.me，半圓指針儀表顯示 0–100 數值與分級文字 |
| **F-16** | VIX 儀表 | 呼叫 Finnhub 取 VIX，顯示數值；失效顯示「暫無資料」 |
| **F-17** | CNN 美股恐懼貪婪儀表 | 呼叫 CNN 非官方端點，顯示數值與分級；**失效時顯示「暫無資料」且不影響其他儀表** |
| **F-18** | 總經數據儀表 | 呼叫 FRED 取 CPI、Fed 利率、DXY、10Y 殖利率、失業率最新值，逐項顯示數值與資料日期；個別項失效顯示「暫無資料」 |
| **F-19** | admin 登入 | `/admin` 未登入導向登入頁；以環境變數帳密驗證；成功後設 session cookie（約 7 天） |
| **F-20** | admin 標的管理 | 登入後可新增/停用標的（含選類別、TradingView symbol、是否有圖）；寫入前驗證 admin session；變更後 `GET /api/watchlist` 反映變更 |
| **F-21** | 每日排程入口 | `POST /api/cron/daily` 驗證 `CRON_SECRET` 後依序執行 F-04、F-07~F-10；Vercel Cron 設台北 09:00 |
| **F-22** | 每週排程入口 | `POST /api/cron/weekly` 驗證 `CRON_SECRET` 後執行 F-11；Vercel Cron 設週一台北 09:00 |
| **F-23** | 資料保留清理 | 排程清除 `daily_digest`/`price_snapshot` > 90 天、`weekly_digest` > 3 年的資料 |

### 反例規格（這軟體不該做什麼）

- 新聞摘要**不該**逐字翻譯或整段照抄原文——必須是 Claude 用自己的話重寫的繁中摘要。
- 摘要**不該**包含買賣建議或目標價——只解讀為何漲跌。
- 價格牆漲跌%**不該**在白天隨現價跳動——它是 9:00 定格值，一天只在 09:00 更新一次。
- 美股價格在休市時**不該**顯示一個會誤導的「0%」或舊漲跌——應明確顯示「休市」。
- admin 密碼**不該**出現在任何 log、前端 bundle、git 歷史中。
- CNN 情緒端點失效時**不該**讓儀表頁整頁報錯或無限載入——僅該儀表顯示「暫無資料」。
- watchlist 寫入 API**不該**接受未通過 admin 驗證的請求，即使請求宣稱來自前端。
- 摘要生成**不該**在使用者每次開首頁時即時呼叫 Claude——只在每日/每週排程時生成一次，前端讀快取。

### 資料與資安要點

1. **處理什麼資料**：公開市場資料（價格、新聞、指數）、AI 生成的摘要、一組自選標的清單、一組 admin 帳密。**不處理任何個人財務、持倉、付款、PII 資料。**
2. **資料敏感度**：市場資料與摘要為公開等級（全站唯讀公開）。唯一敏感資產是 **admin 帳密、各 API key、CRON_SECRET、Supabase service role key**——這些屬機密，只能存環境變數、只在 server 端使用。
3. **AI 不可接觸什麼資產**：AI（Claude Code）開發時**不可**將任何真實 key/密碼寫進程式碼、貼進對話或 commit 進 git；不可在前端 bundle 暴露 service role key 或任何 server-only key。執行階段的 Claude Sonnet 摘要呼叫**不可**被餵入任何機密（它只該收到公開新聞文字）。
